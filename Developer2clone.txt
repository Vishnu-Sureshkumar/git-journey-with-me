2️⃣ Clone Repository into Each Folder

cd dev1
git clone https://github.com/Vishnu-Sureshkumar/git-journey-with-me.git .

Developer 2:

cd ..
cd dev2
git clone https://github.com/Vishnu-Sureshkumar/git-journey-with-me.git .

The . at the end makes sure Git puts repo files directly inside the folder, no nested folder.

Key Issues & Solutions
Issue	Cause	Solution
Second clone does not see branch	Branch exists only locally in first clone	Push branch to GitHub first (git push origin <branch>)
Cannot checkout remote branch directly	Local branch does not exist yet	git checkout -b <branch> origin/<branch> or git fetch origin + git checkout <branch>
Nested folder when cloning	Default clone creates folder with repo name	Use git clone <url> . inside empty folder
Local changes not visible in another clone	Changes not pushed	Commit + push to remote, then pull in other clone
8️⃣ Tips / Notes

Always check remotes:

git remote -v

Always check branch tracking:

git branch -vv

Use branches to simulate multiple developers

Use git fetch before checking out remote branches

Use git pull to sync changes from others
