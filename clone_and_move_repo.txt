Topic: Cloning a GitHub Repository and Moving it to Desktop

'''
Location: C:\Users\Vishnu.S
- git clone https://github.com/Vishnu-Sureshkumar/git-journey-with-me.git

Location: C:\Users\Vishnu.S
- cd git-journey-with-me

Location: C:\Users\Vishnu.S\git-journey-with-me
- move git-journey-with-me Desktop   (error because we were inside the folder)

Location: C:\Users\Vishnu.S\git-journey-with-me
- cd ..

Location: C:\Users\Vishnu.S
- move git-journey-with-me Desktop

Location: C:\Users\Vishnu.S
- cd Desktop

Location: C:\Users\Vishnu.S\Desktop
- cd git-journey-with-me

Location: C:\Users\Vishnu.S\Desktop\git-journey-with-me
- git status
'''


Objective:
Understand how to clone a remote repository and manage folder locations in the local system.

Step 1: Open VS Code Terminal
Opened terminal in VS Code.

Initial location:
C:\Users\Vishnu.S>

Step 2: Clone the GitHub Repository

Command used:

git clone https://github.com/Vishnu-Sureshkumar/git-journey-with-me.git

Purpose:
This command downloads the remote GitHub repository to the local machine.

Result:
Git created a new folder called:
git-journey-with-me

inside:
C:\Users\Vishnu.S

Output observed:

Cloning into 'git-journey-with-me'...
Receiving objects: 100%, done.

Meaning:
All files from GitHub were successfully copied to the local system.

Step 3: Move Inside the Repository

Command used:
cd git-journey-with-me

Purpose:
To enter the repository folder and start working with Git.

Current location became:

C:\Users\Vishnu.S\git-journey-with-me>

Step 4: Attempt to Move Repository to Desktop

Command used:

move git-journey-with-me Desktop

Error received:

The system cannot find the file specified.

Reason:
We attempted to move the folder while already being inside it.

Git folder structure example:

C:\Users\Vishnu.S
    git-journey-with-me
        .git
        README.md

When inside the folder:

C:\Users\Vishnu.S\git-journey-with-me>

The system cannot move the same folder because the command tries to find another folder with the same name inside it.

Step 5: Correct Solution

First move one level up using:

cd ..

New location:

C:\Users\Vishnu.S>

Then move the folder:

move git-journey-with-me Desktop

Result:
The repository folder was successfully moved to:

C:\Users\Vishnu.S\Desktop\git-journey-with-me

Step 6: Navigate to New Location

Commands used:

cd Desktop
cd git-journey-with-me

Current working directory:

C:\Users\Vishnu.S\Desktop\git-journey-with-me>

Step 7: Verify Git Repository

Command used:

git status

Output:

On branch main
Your branch is up to date with 'origin/main'.

Meaning:
The Git repository is working correctly even after moving the folder.

Key Learning Points:

1. git clone downloads a remote repository to the local system.
2. Git repositories can be moved like normal folders.
3. You cannot move a folder while inside it.
4. Use "cd .." to go to the parent directory before moving a folder.
5. git status helps verify that the repository is still functioning correctly.
